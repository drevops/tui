# Feedback

Collection sometimes triggers slow work: a discovery scan of a target directory, a dynamic option set that has to be computed, a derived value resolved against external state. Left silent, the form looks frozen. Two feedback primitives on the facade wrap that work and show it is running - a spinner when the length is unknown, a progress bar when the number of steps is known.

Both wrap a callback and pass its return value straight back, so they drop in around work you already have. The callback receives the primitive, so it can drive the animation as it goes. Both honour the facade's colour and Unicode switches, fall back to an ASCII glyph set when Unicode is off, and - where animation would only make noise - stay quiet: off a TTY, or in headless collection, they print the caption once as a plain line and emit no control sequences at all.

## Spinner

An indeterminate spinner animates a glyph beside a caption until the callback returns. `tick()` advances it one frame; a callback that never ticks simply shows a steady first frame:

```php
use DrevOps\Tui\Feedback\Spinner;

$baskets = $tui->spinner('Counting the baskets', function (Spinner $spinner): array {
  $counted = [];

  foreach (['Apple', 'Pear', 'Plum', 'Cherry'] as $fruit) {
    $counted[] = $fruit;
    $spinner->tick();
  }

  return $counted;
});
```

Runnable in [`playground/15-feedback-spinner.php`](https://github.com/drevops/tui/blob/main/playground/15-feedback-spinner.php).

## Progress bar

When the number of steps is known up front, a determinate progress bar fills as it advances, the counterpart to the spinner for work of known length. Each `advance()` moves the bar one step and can replace the trailing label:

```php
use DrevOps\Tui\Feedback\ProgressBar;

$items = ['Apple', 'Carrot', 'Tomato', 'Spinach'];

$tui->progress(count($items), 'Packing the order', function (ProgressBar $bar) use ($items): void {
  foreach ($items as $item) {
    $bar->advance('packed ' . $item);
  }
});
```

Runnable in [`playground/15-feedback-progress.php`](https://github.com/drevops/tui/blob/main/playground/15-feedback-progress.php).

## Degrading off a TTY

Feedback is chrome, not data, so it is drawn on standard error and animates only when standard error is an interactive terminal. Piped, redirected or collected headlessly, both primitives print the caption once as a plain line and emit no cursor, colour or carriage-return sequences, so a captured log stays clean:

```bash
php playground/15-feedback-progress.php 2>&1 | cat
# Packing the order
```

The colour and Unicode switches are the facade's own (`->color(...)`, `->unicode(...)`), so `NO_COLOR=1` drops the colour and a non-UTF-8 locale (`LC_ALL=C`) swaps the block and braille glyphs for an ASCII bar and `|/-\` spinner frames.
