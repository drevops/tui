# AI agents

A form built with this engine is self-describing: it hands an AI agent (or any automation) the questions, their allowed values and the precedence rules, so answers arrive unattended without the agent ever reading the form's source. The facade exposes three calls for this - `agentHelp()`, `schema()` and `validate()` - which a consumer surfaces in its own tool so an agent can discover the form the moment it meets it.

## The agent guide

`agentHelp()` returns a plain-text guide to the whole form: how to answer it, the precedence order, and every question with its id, type, requiredness and bounds. There is no flag to wire up on the library side - the consumer retrieves the text and folds it into its own help, wherever that belongs (an "AI agents" section of `--help`, a dedicated flag the consumer adds, a generated README):

```php
use DrevOps\Tui\Tui;

// The library hands back the text; the consumer decides where it goes.
echo (new Tui($form))->agentHelp();
```

The guide the produce-order form emits:

```text
Drive the form non-interactively:

- Pass --no-interaction to resolve every question from defaults, discovery and derivation without prompting.
- Pass --prompts with a JSON object (or a path to a JSON file) of answers keyed by question id; these take the highest precedence.
- Set per-question environment variables named TUI_<ID> (the uppercased question id); these win over discovery but lose to --prompts.
- Precedence: --prompts > environment > discovered > derived > default.

Questions:
  name [text] (required) - Order name
  fruit [select] - Fruit
  quantity [number] - Quantity (between 1 and 99)
  organic [confirm] - Organic only?
```

The `--no-interaction` and `--prompts` the guide names are the [headless interface](/headless-collection) a consumer maps onto `collect()`, not flags the library ships. The `TUI_` prefix on the environment line is the default; change it with `->envPrefix('MYAPP_')` on the form or `new Tui($form, env_prefix: 'MYAPP_')`, and the guide reflects the active prefix. A `select` shows only its label here, not its allowed values - those come from the schema below.

## Machine-readable schema and validation

`schema()` returns the questions as structured JSON - id, type, options, bounds and requiredness per question - the machine-readable counterpart to the text guide, and the source an agent reads for a `select`'s allowed option values. A consumer surfaces it the same way, as help output or behind a flag it chooses:

```json
{
  "prompts": [
    {
      "id": "name",
      "type": "text",
      "label": "Order name",
      "description": "",
      "options": [],
      "default": "",
      "required": true,
      "min": null,
      "max": null,
      "step": null,
      "min_date": null,
      "max_date": null,
      "week_start": null,
      "when": null,
      "derive": null,
      "discover": null,
      "depends_on": []
    }
  ]
}
```

`validate()` checks an answer set against those same rules before collection, so an agent can confirm a payload without running the form. Each violation is one message; an empty list means the answers are valid:

```php
$errors = $tui->validate(['name' => 'Weekly Box', 'fruit' => 'grape', 'quantity' => 500]);
// [
//   'Question "fruit": value "grape" is not one of: apple, banana, cherry.',
//   'Question "quantity" must be between 1 and 99.',
// ]
```

## Precedence and environment variables

Once an agent supplies answers through the consumer's [headless interface](/headless-collection), every field resolves from the first source that provides a value, in this order:

1. `--prompts` - the JSON payload (a string or a path to a JSON file), keyed by question id. Highest precedence.
2. **Environment** - a per-question variable named `<PREFIX><ID>` (the uppercased id), e.g. `TUI_NAME`. The prefix defaults to `TUI_`.
3. **Discovered** - values detected from the target directory (see [Discovery](/discovery)).
4. **Derived** - values computed from other fields.
5. **Default** - the declared default.
